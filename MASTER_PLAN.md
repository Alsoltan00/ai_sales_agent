==============================================================
1. واجهة تسجيل الدخول
==============================================================

الوصف:
الشاشة الأولى التي يراها أي مستخدم عند فتح النظام.
تحتوي على زرين رئيسيين:
  - "سجّل الآن"    : فتح شاشة إنشاء حساب جديد
  - "تسجيل الدخول" : مطابقة بيانات الدخول من جداول المستخدمين والعملاء

منطق تسجيل الدخول:
  1. إذا تطابقت البيانات مع مستخدم (موظف) في جدول users:
       → دخول بصلاحيات المستخدم المخصصة → واجهة الإدارة (Admin)

  2. إذا تطابقت البيانات مع عميل/تاجر في جدول clients:
       → دخول بوضع التاجر → واجهة التاجر (Home)

  3. إذا لم تُوجد البيانات في قاعدة البيانات:
       → تحويل إلى واجهة إنشاء حساب جديد

منطق إنشاء حساب جديد:
  1. يُطلب إدخال: اسم الشركة + رقم التواصل
  2. تُرسل البيانات إلى قائمة "العملاء الجدد" وتنتظر الموافقة
  3. يظهر الطلب فقط للمستخدمين الذين يملكون صلاحية "العملاء الجدد"
  4. إذا كان الحساب موجوداً مسبقاً → تحويل مباشر لواجهة تسجيل الدخول

==============================================================
4. إدارة العملاء
==============================================================

4.1 العملاء الجدد:
  - العميل الذي أنشأ حساباً وأدخل اسم الشركة
  - يظهر امامه زر قبول أو رفض يراه فقط من لديه صلاحية "العملاء الجدد"
  - عند القبول  → يُحوّل العميل إلى واجهة إدارة الاشتراكات
  - عند الرفض  → يتحول لون العميل للأحمر مع منع التعديل عليه

4.2 إدارة الاشتراكات:
  - إدارة كاملة لاشتراكات العملاء: تجديد، إيقاف، تعديل

5. إدارة المستخدمين (الموظفين)

  - لكل مستخدم صلاحيات مخصصة تُعرض على شكل جدول
  - الإضافة والتعديل يتم فقط من قِبل من لديه صلاحية الأدمن

6. واجهة الاستقبال
==============================================================

قنوات الاستقبال المدعومة:
  1. Telegram Bot        — كل ما يتطلبه توصيل تيليجرام واستلام الرسائل
  2. WhatsApp Evolution  — كل ما يتطلبه توصيل Evolution API واستلام الرسائل
  3. WhatsApp الرسمي    — كل ما يتطلبه توصيل WhatsApp Cloud API واستلام الرسائل

خوارزمية الاستقبال (5 خطوات):

  الخطوة 1: استلام الرسالة
    → تُحلَّل بواسطة النموذج المحدد
    → يُفهم معناها بشكل ذكي (NLU) وليس مطابقة أحرف فقط

  الخطوة 2: البحث في قاعدة البيانات
    → البحث في جدول بيانات العميل
    → يتم البحث بحسب الشروط الموضحة في واجهة التخطيط

  الخطوة 3: التحقق من النتائج
    → البيانات المستخرجة تُرسل مجدداً للنموذج للتحقق
    → يتحقق من مطابقتها لمعايير الشركة ولما طلبه العميل
    → يُرسل الرد عبر واجهة الإرسال

  الخطوة 4: التعامل مع الطلبات غير المنطقية
    → إذا كان السؤال خارج موضوع الشركة أو غير منطقي
    → يرد باحترام وذكاء مع اتباع الشروط المحددة

  الخطوة 5: التعامل مع عدم توفر البيانات
    → لا يُرسل إلا رد يناسب سؤال العميل
    → إن لم تُوجد البيانات في الجدول → اعتذار بطريقة مناسبة

7. واجهة إدارة المتجر
==============================================================

الحقول المطلوبة:
  - اسم الشركة / المؤسسة  : الاسم الرسمي
  - رقم التواصل            : رقم الواتساب أو التيليجرام (قابل للتعديل)
  - كلمة المرور            : قابلة للتعديل
  - البريد الإلكتروني      : البريد الخاص بالشركة
  - رابط المتجر            : اختياري (قابل للتعديل في أي وقت)

ملاحظة: يستطيع العميل تعديل رابط المتجر ورقم التواصل في أي وقت.

==============================================================
9. واجهة التخطيط
==============================================================

الوصف:
واجهة تحديد "شخصية" الوكيل الذكي وقواعد تفاعله.

الحقول المطلوبة:
  1. الوصف:
       وصف دقيق للشركة يُستخدم عند طلب العميل مواصفات الشركة

  2. اللهجة والترحيب:
       إدخال اللهجة التي سيتكلم بها النموذج
       مع أوامر صارمة: 5% فقط للترحيب، الباقي للمبيعات

  3. اسم موظف المبيعات:
       يُستخدم عند استقبال سؤال من نوع "من أنت؟" أو ما يقاربها

==============================================================
10. واجهة تدريب نماذج الذكاء الاصطناعي
==============================================================

آلية العمل:
  1. تُعرض النماذج المشهورة (GPT-4, Claude, Gemini, Groq, ...)
  2. عند تحديد نموذج → تظهر الخانات المطلوبة (API Key, Model ID, ...)
  3. عند النقر على "حفظ":
       - لا يُحفظ إلا بعد التحقق الفعلي من أن النموذج يعمل
       - يُختبر النموذج أولاً ثم يُحفظ
  4. بعد الحفظ الناجح:
       - تظهر علامة خضراء تدل على أن النموذج يعمل
       - تظهر مربعات اختبار للتجربة والتدريب

خصائص واجهة التدريب على الأعمدة:
  - تُعرض الأعمدة المسحوبة من واجهة مزامنة البيانات
  - يمكن كتابة ملاحظة عند كل عمود لشرح طريقة التعامل معه
  - زر إيقاف عمود: لا يُفكّر النموذج بداخله مهما حدث

==============================================================
11. واجهة مزامنة البيانات
==============================================================

مصادر المزامنة المدعومة:
  1. Supabase     → كل 5-10 دقائق (مزامنة مستمرة)
  2. Aiven        → كل 5-10 دقائق (مزامنة مستمرة)
  3. Google Sheets → كل 30 دقيقة (يُدخل رابط الملف + اسم الورقة)

آلية عمل المزامنة:
  1. يُدخل العميل بيانات الاتصال
  2. يُدخل اسم الورقة/الجدول المراد سحبه
  3. أول صف يُعتبر دائماً أسماء الأعمدة (Header Row)
  4. تبدأ المزامنة التلقائية بالتوقيت المحدد لكل مصدر
  5. دائماً يحتفظ النظام بنسخة في قاعدة البيانات الخاصة بالعميل

==============================================================
12. واجهة عرض البيانات
==============================================================

الوصف:
  - عرض مرئي لجميع البيانات التي تمت مزامنتها
  - يمكن الاطلاع عليها ومراجعتها في أي وقت

==============================================================
13. واجهة الأرقام المصرّحة
==============================================================

آلية العمل:
  العميل يسجل رقماً → ينقر "إضافة" → يُضاف للقائمة المصرّحة

أزرار التحكم الإضافية:
  - "عدم الرد على أي مجموعة" : إيقاف الرد على رسائل المجموعات
  - "السماح بالرد على الكل"  : رفع جميع القيود

قاعدة الأمان الرئيسية:
  إذا وصلت رسالة من رقم غير مضاف في هذه الواجهة:
    × لا يتم إرسالها
    × لا يتم تحليلها
    × لا يتم التعامل معها بأي شكل

14. خوارزمية الذكاء الاصطناعي — منطق الرد الآلي الكامل
==============================================================

    ↓

    └─ سؤال خارج الموضوع    → رد احترامي ذكي
    ↓
[إرسال عبر القناة المحددة: Telegram / WhatsApp Evolution / WhatsApp Official]

15. هيكل قاعدة البيانات — الجداول المطلوبة
==============================================================

جدول: users
  - id, name, email, password_hash, role, permissions (JSON), created_at

جدول: clients
  - id, company_name, contact_number, email, password_hash, store_url,
    status (pending/active/rejected), created_at

جدول: subscriptions
  - id, client_id, plan, start_date, end_date, status (active/expired/paused)

جدول: new_client_requests
  - id, company_name, contact_number, status (pending/accepted/rejected),
    reviewed_by, reviewed_at, created_at

جدول: authorized_numbers
  - id, client_id, phone_number, label, added_at

جدول: ai_models_config
  - id, client_id, model_name, api_key, model_id, is_active,
    tested_at, is_working (bool)

جدول: sync_config
  - id, client_id, source_type (supabase/aiven/google_sheets),
    connection_details (JSON), table_name, sheet_name, sync_interval_minutes

جدول: planning_config
  - id, client_id, company_description, dialect_instructions,
    sales_agent_name, created_at, updated_at

جدول: column_training
  - id, client_id, column_name, note, is_disabled (bool)

جدول: message_logs
  - id, client_id, channel, direction (in/out), phone_number,
    message_text, ai_response, timestamp
